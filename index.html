Possivelmente, a forma mais fácil de criar um site hoje em dia é usando GitHub Pages (talvez Google Sites seja mais fácil, mas aí não é você quem está fazendo o trabalho.) Você cria um repositório no GitHub com o nome username.github.io coloca seu código lá e você já tem um site no ar. Sem precisar pagar por hospedagem. Nem ter um domínio. Nem nada. Você coloca seu código nesse repositório e tem um site funcionando.

A internet é a evolução de um sistema de organização de informação proposto pelo físico Tim Berners-Lee para resolver o problema de comunicação que os cientistas do CERN enfrentavam na época.

[...] the hope would be to allow a pool of information to develop which could grow and evolve with the organisation and the projects it describes. [...] This is why a "web" of notes with links (like references) between them is far more useful than a fixed hierarchical system. When describing a complex system, many people resort to diagrams with circles and arrows. Circles and arrows leave one free to describe the interrelationships between things in a way that tables, for example, do not. The system we need is like a diagram of circles and arrows, where circles and arrows can stand for anything.
~ Tim Berners-Lee, Março de 1989, Maio de 1990 - Information Management: A Proposal

Por design, documentos html não contêm informação sobre como devem ser renderizados. E a maior parte do trabalho deve ser feito pelo browser. A implementação do navegador NeXT permitia o uso de style sheets para alterar como os documentos seriam renderizados. Muitos padrões para style sheets foram propostos para permitir personalização de documentos como outros programas de preparação de documentos da época, entre estes tex, o precursor do LaTeX, já permitiam. Entre essas propostas figurava a primeira versão de Cascading Style Sheets ou simplesmente CSS.

Em 1995, Brendan Eich desenvolveu, em dez dias, a linguagem javascript para ser usada no navegador Netscape. O plano original era usar Java, mas os gestores do projeto julgaram que uma linguagem de scripts seria mais adequada para a situação. Hoje javascript é suportada por todos os navegadores modernos e em 2009 Ryan Dahl criou o runtime Node.js permitindo que javascript fosse executada fora dos navegadores.

Any application that can be written in JavaScript, will eventually be written in JavaScript.
~ Jeff Atwood

Essas tecnologias hoje são o padrão, e os navegadores são muito mais complicados do que eram antes. O processo de desenvolvimento de um website hoje é geralmente muito mais complexo do que se pretendia na época. Muitas ferramentas foram desenvolvidas para melhorar javascript. Foram criadas linguagens de programação que precisam ser transpiladas para javascript como CoffeeScript e TypeScript e um número comicamente grande de frameworks para essas linguagens. Chegando ao ponto de empresas terem times inteiros dedicados a transpilação e bundling de websites.

A internet hoje é bem complicada. E de alguma forma em todo canto que se olha tem alguém dizendo que ela é fácil, estranhamente, essas pessoas vão te ensinar tudo que você precisa saber em algum curso. Com cada tecnologia adicionada com a intenção de melhorar javascript a complexidade do sistema aumenta, e no final, nem os desenvolvedores sabem mais como as coisas funcionam. O instrutor do curso não sabe. E muito menos o aluno depois de assistir cinco horas de aulas em vídeo.

Un auteur ne nuit jamais tant à ses lecteurs que quand il dissimule une difficulté.
~ Évarist Galois

Existem movimentos que tentam reagir a tanta complexidade, notavelmente o suckless é um deles. Os programas que eles desenvolvem são simples, e precisam ser compilados pelo usuário. O que assusta demais pra mim.

Minha intenção com esse website é ser tão simples quanto possível. Meu objetivo não é construir o que se chama por aí de web application, mas sim um website pessoal resgatando o espírito dos primórdios da internet.

Eu pretendo escrever sobre matemática e sobre essa minha jornada tentando fazer sentido da tecnologia moderna. Estou lendo o livro The C Programming Language no momento, mais como um hobby nessa missão de entender como software realmente funciona.
